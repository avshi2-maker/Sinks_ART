## ⚠️ LESSONS THAT COST TIME TODAY (do not repeat)

1. **WRONG PAGE, not wrong code.** Avshi looked for the sketch-gallery button on `/sketch` (the builder)
   for a long time — but `DemoCard` only renders on `/demos` (the gallery). When a feature is "missing,"
   FIRST confirm the URL/page it actually lives on, before touching code. (`/sketch` already had its own
   צור הזמנת ייצור + שלח לאלס buttons — different component.)
2. **Unsaved sketch = gone.** A sketch built in `/sketch` lives ONLY in the browser until שמור לגלריה is
   clicked (writes to `demo_trials`). The morning Goldman sketch was never saved → had to rebuild. There is
   no autosave. (Possible future task: warn-on-navigate-away, or autosave draft.)
3. **DELIVER FILES AS DOWNLOADS, NOT INLINE.** Avshi's hard rule (stated forcefully today): NEVER paste giant
   base64/inline file writes into chat. Build the file, put it in outputs, give a download link + a short
   PowerShell move block. Inline base64 also risks char corruption (a stray char broke one payload today).
4. **Commit order matters (Rule #31 sibling).** DemoCard used `project_title` but `demosData.ts` (which defines
   it) hadn't been moved to disk yet → `next build` type error. Deliver the file that DEFINES a type before/with
   the file that CONSUMES it. When a build error says "Property X does not exist," check the defining file is on disk.
