### HARD-WON LESSONS FROM TODAY (do not repeat these mistakes)
1. **NEVER wrap a file in a PowerShell single-quoted here-string when it contains apostrophes.** It mangled a whole TSX file into 300+ fake errors. **Instead: deliver as a downloadable file + Move-Item, or base64-encode and decode on his machine** (`[System.Convert]::FromBase64String(...)` вЖТ `[System.IO.File]::WriteAllBytes(...)`). The base64 path is bulletproof вАФ use it for any file with Hebrew/quotes.
2. **`.Replace()` with multi-line CRLF anchors keeps returning 0 matches.** Always provide a regex fallback with `[\s\S]*?` / `\s*` and check `([regex]::Matches($c,$pattern)).Count` first. Single-line anchors are far more reliable.
3. **DON'T GUESS вАФ READ THE FILE.** Wasted ~6 turns on the empty-dropdown bug by theorizing (cache, refresh, archived_at) when the actual cause was the page never passed the `customers` prop to the component. When something's wrong, `Get-Content` the real file and the pageвЖТcomponent wiring BEFORE proposing a cause.
4. **Server pages snapshot data on load.** Adding a row in another tab won't show until the page re-fetches. `export const revalidate = 0` + `dynamic = 'force-dynamic'` + `router.refresh()` after a save.
5. **DOCX RTL is a known time-sink вАФ abandoned on purpose.** Word opens LTR despite bidi injection. We pivoted to **RTL copy-text вЖТ paste into Gmail/Outlook via Notepad**, which Avshi confirmed works. Do not re-enter the Word RTL rabbit hole.
